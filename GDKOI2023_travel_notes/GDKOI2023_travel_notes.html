Problem 1

题目大意：

给出三个 n×n 的矩阵 A 、 B 、C 。求 A×B 是否等于 C 。有多组询问。

数据范围：

对于 20% 的数据，∑n≤300 。

对于另外 20% 的数据， Ai,j≠0 的不超过 n 个。

对于 100% 的数据，∑n≤3000 。

T1 看起来很不错啊，题目通俗易懂，简单明了。暴力很好打，可惜只有 20 分。目光对准 20 分的性质，发现此时可以枚举有数值的 Ai,j ，考虑它对答案的贡献。由矩阵乘法定义式：

Ci,j=Ai,k×Bk,j

枚举 i,k 。如果 Ai,k 不为 0 ，就枚举 j 来贡献，这样是 O(n2) 的。

最后我觉得差不多了，感觉有机会想出正解，但是感觉又不一定想出来。于是就快速把 40 分的部分分打出来了。

打完以后又打了一个对拍，拍了一下特殊性质和暴力，结果是对的，就决定从特殊性质出发，考虑能否想出正解。

考虑将 O(n3) 的算法优化一维，就考虑到题目只是问是否相同，并没有让我将结果矩阵求出来，考虑可不可以用奇妙的办法将 j 优化掉。

于是想到似乎 Ai,k 会乘上 B 的某一行贡献到 C 的某一行，一开始想的是：Ai,k 是固定的，不如对 B 整行求和，乘上 Ai,k ，看是否等于 C 的行。但是这样如果每一行和是相同的，但具体数字不一样就会出错。

考虑将 B 不同的列乘上随机不同的数字，这样就可以将数字之间的差异放大化。打完对拍，似乎能过。

最后生成了大小为 3000 的一组数据，发现文件有一百多M，读入有点慢，跑了 5s 多才跑完，还开了O，有点怕 T 。

两个小时过去了。

Problem 3

题目大意

给定一张 n 个点 m 条边的无向图和一个长度为 n 的数组 a1,a2,⋯,an 以及一个整数 C ，你需要求出有多少个长度为 n 的数组 b 满足：

  1. 0≤bi≤ai， ∀1≤i≤n。

  2. 对于每条边 (u,v)， bu≠bv 。

  3. b1⊕b2⊕⋯⊕bn = C，其中 ⊕ 代表异或。

答案对 998244353 取模。

本来考虑异或的特殊性质，想到是否可以对于 b 的每一位计算这一位为 C 的概率，推了一会，发现第二个限制有点难办。

Problem 1

题目大意

给定一棵 n 个结点的树，有 Q 次询问，每次给定 x,y,z，你要找到三个点 (u,v,w) 满足 dis(u,v)=x,dis(u,w)=y,dis(v,w)=z。其中 dis(u,v) 表示树上 u 和 v 两点唯一简单路径所包含的边数， dis(u,u)=0。 保证有解。

我以为今天的 T1 还像昨天的那么水，想了一会，观察了一会样例，发现似乎给出的 x,y,z 中最小的两个相加等于最大的两个，想到如果最小的两个数加起来为最大的，那么三个点必定在一条链上，那么只需要在最长的链（即直径）上找两个点就好了。打树上倍增，在 O(qlog⁡n) 的时间内解决问题。

打之前我还有点疑惑，真的有这么简单吗？最小的两个数加起来真的等于最大的数字吗？但是我又举不出任何反例，自信应该是这样的。大不了打完之后打对拍就好了。

题目大意

给定一棵 n 个结点的有根树 T，结点从 1 开始编号，根结点为 1 号结点，每个结点有一个正整数权值 vi。有 Q 次询问，对于一次询问，给定 (x,k)，设 x 号结点的子树内（包含 x 自身）的所有满足距离 x 号结点不超过 k 的结点编号为 c1,c2,⋯,ck，则这次询问的答案为：

Day 4

早上想了一想，想出来了。

跟 LYM 讨论了一下，终于想出了正解。

还是这张图。如果知道点 i 的儿孙中的最大深度 ai ，次大深度 bi ，第三大深度 ci 。 满足 ai≥A 且 bi≥B 且 ci≥C ，那么可以用树上倍增就可以在 O(logn) 的时间里解决一次询问。

考虑如何解决，用出题人的想法，用树状数组。将 ai 和 A 排序，倒序处理。对于每一次询问，很显然，对于所有 i 满足 ai≥A 的，以 bi 为下标，将 ci 加入树状数组，利用树状数组询问后缀最大值，就可以快速求出 i 了。利用树上倍增就可以在 O(nlog⁡n) 的时间内解决问题了。

总结

  • 难的比赛一定要注意部分分，有时候 10 分就可以相差很多名次。不能抛弃任何可能的部分分。

  • 打之前一定要考虑清楚，不能想个大概就打，很容易出问题。高难度的比赛题目往往结论并不那么简单，可能设计一些比较难的结论，一定要保证结论正确。

  • 我和一些优秀的同学之间的差距还很大，要多花一点时间在学习信息学上面，多学多做题，争取赶上其他同学。

  • Day 1 T3 TLE 了，其实不应该枚举完再检验，应该一边枚举一边检验。可以减去很多情况。
